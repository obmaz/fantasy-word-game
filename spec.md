# 판타지 영단어 게임 (Fantasy Word Game) 소프트웨어 개발 상세 명세서 (SRS)

이 문서는 시스템을 새로 구축할 때 사용하는 완벽한 개발 지침서입니다. 프론트엔드/백엔드 언어나 프레임워크 선택에 구애받지 않고, **이 명세서와 리소스 폴더(`images/`, `data/`)만 있으면 기존과 정확히 동일한 기능을 하는 클론 게임을 개발할 수 있도록** 매우 구체적으로 작성되었습니다.

---

## 1. 게임 시스템 아키텍처 및 설정

### 1.1 뷰포트 및 레이아웃 제약 사항
- **해상도 비율**: 9:16 모바일 세로 뷰 최적화. 데스크톱 접속 시 화면 중앙에 고정된 모바일 뷰 배율로 표시.
- **주소창 대응(App Height Lock)**: 모바일 브라우저 진입 시점의 `window.innerHeight`를 계산하여 레이아웃 최상단 컨테이너의 높이(height)로 영구 고정합니다. (모바일 브라우저의 동적 주소창 숨김/나타남에 따른 UI 출렁거림 방지)
- **컴포넌트 스케일링 체계**: 미디어 쿼리 방식 대신, CSS 컨테이너 쿼리 단위(`cqmin`)와 뷰포트 넓비 대비(`vh`/`vw`)의 동적 스케일링을 보장하는 **`clamp(min, 동적크기, max)`** 함수를 사용하여 모든 디자인 컴포넌트 크기가 유기적으로 조절되도록 제어해야 합니다.
- **글로벌 Z-Index 계층 (Layers)**:
  - `Level 0 (Background)`: 각 모드별 배경 이미지.
  - `Level 1 (Content)`: 주 콘텐츠 영역 (인게임 뷰, 몬스터, 플래시 카드 영역 등).
  - `Level 2 (Controls)`: 화면 하단의 퀴즈 패널, 옵션 버튼 등 메인 조작부.
  - `Level 3 (Modal)`: 서브 팝업 창 (전투 준비 단계, 상점, 가방, 시스템 팝업 등).
  - `Level 4 (Overlay)`: 뮤직 오버레이, 재화 정보 창, 최상위 핫키 메뉴.

### 1.2 에셋(Asset) 디렉토리 구조 맵핑 (파일별 사용 용도 상세)
- **오디오 (`data/`)**
  - `background_music_1.mp3` ~ `23.mp3`: 각 스테이지나 테마별로 재생되는 글로벌 배경음악 루프 파일로, 음악 선택 UI 또는 스테이지 진입 시 연속 재생됩니다.
- **타이틀 (`images/title/`)**
  - `title.webp`: 타이틀 화면의 메인 배경 이미지로, 화면에 꽉 차게 렌더링됩니다.
  - `title_header_{1~4}.webp`: 타이틀 상단 로고 이미지 4종으로, 앱 구동 대문 진입 시 무작위로 1개가 선택되어 표시됩니다.
  - `practice/battle/boss/shop/inventory/statistics/settings` 전용 버튼 이미지: 타이틀 메뉴에서 각각의 모드 및 팝업을 호출하는 개별 아이콘 버튼 스킨 이미지로 사용됩니다.
  - `apply_btn.webp`, `exit_btn.webp`: 설정 적용 완료, 모달 뒤로가기/닫기 등 전반적인 시스템에서 공통으로 쓰이는 UI 아이콘 에셋입니다.
- **연습 모드 (`images/practice_mode/`)**
  - `background.webp`: 연습 모드 인게임 진입 시 뒤에 깔리는 바탕 배경 뷰입니다.
  - `practice_mode_select_popup.webp`: 연습장 진입을 위한 지역 선택, 난이도 조작 등 준비 조작 창의 배경 모달 스킨으로 사용됩니다.
- **배틀 모드 (`images/battle_mode/`)**
  - `background.webp`: 메인 전투 화면 뒤에 배치되는 고정 전장 배경 이미지입니다.
  - `battle_mode_story_modal.webp`: 스토리 진행이나 특별 이벤트 조우 시 나타나는 텍스트 알림 팝업의 배경 기판으로 쓰입니다.
  - `hero.webp`: 게임 내에서 플레이어의 기본 스탠딩 베이스 모델이 되는 영웅 이미지 파일입니다. 인벤토리에서 장착한 아이템(무기, 장갑 등)이 이 이미지 위에 추가 레이어로 중첩되어 조합 및 합성 렌더링 용도로 사용됩니다.
  - `monster_{1~3}.webp`: 웨이브나 스테이지 진행도에 따라 교체되며 출현하는 적 몬스터 이미지 파일들입니다. 몬스터가 플레이어의 공격을 받아 피격당하면 해당 이미지 위로 플로팅 데미지, 이펙트가 오버레이되는 식의 타겟 마네킹 용도로 사용됩니다.

## 2. 모바일 및 오디오 시스템 로직

### 2.1 글로벌 오디오 재생 규칙 및 음악 선택 UI
- **음악 선택 (Music UI) 컴포넌트 구조**:
  - 인게임 상단(`left: 50%` X축 앵커링) 혹은 타이틀 화면 우상단 등 접근성이 높은 곳에 오버레이(`z-index 4`) 형태로 배치.
  - 재생/일시정지 버튼(`▶️`/`⏸️`), 이전/다음 곡 넘김 버튼 제어 가능.
  - 선택 드롭박스를 통해 클릭 시 1번~23번 트랙 목록 메뉴가 펼쳐짐. 클릭하면 즉시 `mp3`파일을 비동기 스왑하며 재생.
- **백그라운드 제어 규칙**: 
  - 앱 시작 후 잠금 해제된 음악 목록 중 하나를 무작위 재생(Loop 시작). 단 설정 모달에서 꺼져있다면 UI만 동기화하고 정지 상태 유지.
  - 플레이어가 풀이하지 않은 미해금 스토리에 종속된 곡은 콤보박스 디자인에서 `disabled` 제약 사항이 들어가 선택 불가. 

### 2.2 디바이스 피드백 (Vibration & Shake)
- 플레이어가 오답(데미지 판정 골인)을 선택할 경우 즉각 발동:
  1. 기기 진동 API(`navigator.vibrate(200)`)를 통해 물리적 피드백 제공.
  2. 컨테이너 래퍼(`.battle-arena`) 위치에 0.4초간 css `screen-shake` 애니메이션을 연동함.

## 3. 메인 화면 및 주요 모달 상세 명세

### 3.1 타이틀 화면 (Title Screen)
앱 구동 시 가장 먼저 보이는 진입점입니다. 배경 이미지(`title.webp`)의 비율에 맞춘 **절대 상대값(% 기반의 absolute 레이아웃)** 을 이용하여 화면 크기가 바뀌어도 버튼 위치가 오차 없이 고정됩니다.

- **UI 레이아웃 및 앵커 좌표**:
  - 상단 로고: `left: 50%`, `top: 13%`, 로고 4개 중 접속 시 1개를 무작위 렌더링.
  - 스테이터스 UI: 골드 UI는 `top: 3%`, 좌측에서 마진 분산 조절. 우상단 단어장 교체용 콤보 박스에는 고를 때 현재 선택된 단어북의 이름이 나타나야 하며, `right: 3%`에 `width: 50vw`로 위치하여 긴 제목이 축약 처리됨. 
  - 메인 시작 버튼 그룹 (`width: 67%`, 화면 가로 정 중앙): 버튼은 다른 레이아웃 없이 이미지(`practice_btn.webp`, `battle_btn.webp`, `boss_btn.webp` 등)로만 구성됩니다. `연습모드`(`top: 47%`), `배틀모드`(`top: 59%`), `보스모드`(`top: 71%`) 순서로 세로 배치되며, 각 버튼 이미지 위에 해당 모드 글자가 직접 오버레이되어 텍스트로 표현되어야 합니다.
  - 시스템 관리 아이콘 하단 바: 높이 `top: 83%` 선에 가로 일렬 나열, 차례대로 설정(`20%`), 상점(`40%`), 인벤토리(`60%`), 통계(`80%`).
- **기능 로직**: 
  - 상단의 [데이터셋 선택 드롭박스] 값 변경 시 SPA에서 사용할 `game-data` 전역 데이터 상태 값이 즉시 교체 렌더링.

### 3.2 게임 지역 선택 모달 (진입 흐름)
- **연습장 개설**: 모험 지역(Day) 전체와 `All` 옵션을 고를 수 있으며, 난이도는 `10`, `20`, `all` 옵션 존재.
- **배틀 전장 진입**: 모험 지역 및 난이도는 위와 같으며, 추가로 객관식(Objective), 혼합방식(Mixed, 50% 비율 교대 생성), 주관식(Subjective) 라디오 버튼 옵션 포함.
- **보스 모드 개방**: 준비 모달 없이 버튼 클릭 시 즉시 그 데이터셋의 모든 문제를 **무한 모드 주관식 난투** 상태로 진입.

### 3.3 상점 (Shop) & 인벤토리 (Inventory) 관리부
- **[상점 UI 및 로직]**
  - 헤더 뷰 하단으로 3개의 분류(무기/방어구/소모품) Tab이 적용되며 Grid 형태로 아이콘 카드 출력.
  - 플레이어 재화 연산 검수가 통과 시, 영구 장비는 `OwnedItems`에 귀속되어 가격 텍스트가 `[장착하기]` 로 영구 변환. 황금장갑/힌트 등의 소비품은 덧셈 연산하여 중첩 스택을 쌓음.
- **[인벤토리 UI 및 로직]**
  - 좌측 상단 모달 뷰포트에 '플레이어 인체 아바타 다이어그램' (머리, 양손, 발) 슬롯 존재.
  - 우측 하단 뷰포트에 '장착중이 아닌 보유 장비'인 `Storage창고` 썸네일 노출.
  - 창고에서 썸네일을 터치 시 귀속 부위(Slot type) 검수 후 빈 슬롯이라면 즉시 다이어그램에 레이어 오버레이로 장착되며 인게임 액터 모델링에 실시간으로 데이터 Sync 바인딩 됨.

### 3.4 통계 (Statistics) & 설정 (Settings) 제어부
- **[통계 UI 및 로직]**
  - 퍼센티지 Progrcess Bar 대시보드를 차용. `localStorage`의 `playerStats`에 누적되어 온 정답률 데이터를 객체 파싱해 옴. (모달 로드 시 0부터 Target 넘버까지의 카운트 업 애니메이션 동작 지원)
  - 보스모드 최고 랭킹 및 피격되지 않고 퍼펙트 클리어한 지역 뱃지 등의 수집 로그 지원.
- **[설정 UI 및 로직]**
  - 음악 자동 재생 스위치, 효과음 토글, TTS 옵션 등을 관리. 컴포넌트 터치 시 즉각 해당 음소거 API 등 엔진 내부 상태로 하드 Hook 전달.
  - 최하단 구역에 게임을 갈아엎는 `데이터 포맷 리셋 버튼` 존재로 이중 경고 모달 채택. (빈 공간을 연속 타격하여 나오는 관리자 비밀번호 창에서 올바른 키 입력으로 골드 무한 혜택 발동하는 이스터에그 내재)

## 4. 인게임(In-Game) 환경 및 전경 명세

두 모드 모두 컨테이너 Flex를 이용하여, **상단 비주얼 아레나(`flex: 0 0 40%`) / 하단 UI 데스크(`flex: 1`)**라는 황금 분할 룰을 고정적으로 채택해야만 합니다.

### 4.1 배틀 타워 인게임 (`battle-mode-game`)
- **전술 아레나 뷰 (상단 40%)**:
  - `background.webp` 가 깔려있으며, 좌우 구석 `top: 10px` 앵커링을 통해 스테이지 이름, 웨이브 표시기, 카운트다운 타이머 바 부착 배치. 타이머 아래에 골드 표시판 동기화.
  - 하단에 기준축 정렬하여 영웅(`hero-wrapper`) 위치. `16.67vh` 크기로 제한되어 무기 장비를 Absolute로 합성. 반대편에는 웨이브 번호에 매칭된 몬스터 이미지. 데미지 폰트는 공중에 플로팅 텍스트 처리. 전장 위로 칼날 베기(`slash`) 이펙트 컨테이너 레이어 구동.
- **퀴즈 제어부 (하단 flex 영역)**:
  - 우측 코너를 정렬 축(`justify-content: flex-end`) 삼아 힌트 및 필살기 스킬 바 노출.
  - **객관식 문항**: 2x2 사이즈 버튼으로 할당. Grid CSS 활용 `gap: 8px` 로 겹침/여백 이슈 없게 제어.
  - **주관식 문항**: 하단 텍스트 입력창이 중앙 노출되며 영어 스펠링 힌트가 가려져 등장함. 우측 붉은색 파괴 버튼 (Submit) 활성화.

### 4.2 연습 모드 인게임 (`practice-mode-game`)
- **수련 아레나 뷰 (상단 40%)**:
  - `background.webp` 고정 바탕. 헤더부 구석에 좌우 Margin 분산으로 단어장 Day 정보 뱃지와 학습 진행도 카운터 나열됨.
  - 화면을 나가는 `종료/Exit` 버튼은 디자인적으로 아레나 우측 바닥 코너로 Absolute 좌표 고정. 
- **단어 퀴즈 하단 데스크 영역 (하단 flex 영역)**:
  - 수직 Column 로 정렬된 레이아웃.
  - 영단어가 굵게 출력되는 최상단 박스 옆에, 원형 모양의 음성 재생/TTS 스피커 버튼 조립 위치.
  - 하단 네비게이션 액션 바는 CSS Grid로 구성되어, 1:1:1 즉 `1fr 1fr 1fr` 3등분 컬럼으로 [이전 기록], [외웠어요], [다음] 버튼이 한 줄을 차지함.

## 5. 핵심 알고리즘 및 엔진 로직

### 5.1 오답 보기 (Distractor) 생성 원리
- 객관식 4개의 버튼 중 정답 1개를 제외한 3개의 오답을 채우는 알고리즘:
  1. 외부 데이터셋 `decoy-words-set.js` 등에 저장된 특정 단어에 특화된(유사하게 생겼거나 의미가 헷갈려 출제 빈도가 잦은) 오답 풀이 존재한다면 최우선으로 이를 차출한다.
  2. 만약 해당 오답 개수가 부족하거나 부재 중이라면, 현재 진입한 데이터셋 전체(Full Vocabulary Data) 배열에서 무작위로 단어를 Pop하여 풀을 채운다.
  3. 무작위 선택된 오답이 우연히 본 정답과 일치하지 않도록, `lowerCase` 및 `trim` 처리하여 중복 여부를 최종 필터링(Set 중복 제거)한다.

### 5.2 게임 루프와 이터레이터 패턴 (Game Engine)
- 상태 관리 객체: `game` (클로저 혹은 클래스 내부 속성)
  - `game.list`: 모달에서 결정된 난이도(count) 개수와 유형(mixed, objective)에 따라 미리 추출 및 셔플링된 문제들의 임시 런타임 배열.
  - `game.idx`: 현재 플레이어가 풀고 있는 `list` 내부 커서 인덱스.
  - `game.nextLevel()`: 플레이어가 정답을 맞추거나(또는 시간이 만료되어 오답 페널티 애니메이션이 끝난 직후) 커서 인덱스를 1 증가시키며 다음 문제의 컴포넌트 렌더링을 호출하는 중앙 무한 루프 통제 펑션.
  - 배열이 소진된 상황 체크 시 승리로 간주. `showEnding(true)` 결산(Result Popup) 함수 호출. (단, 무한 보스모드의 경우 1회분 배열 소진 시 동적으로 셔플링 데이터를 새로 병합 확장하거나 실패하기 전까지 계속 문제를 뽑아오는 큐 구조 적용)

### 5.3 데이터 로딩 아키텍처 구현 방향성
- `data/game-data-1.js` 등과 같이 모듈 외부 파일에 전역 변수 배리언트로 글로벌 `Data` 시트가 존재하게 세팅. 
- 단어장 선택 드롭박스를 변경하면 `loadGameData(setID)` 함수 내부에서 DOM 조작을 통해 새로운 `<script src="">` 태그를 `document.head`로 새로 주입하여 기존 메모리 상의 데이터를 SPA 구조에서 런타임 갱신함. 치환이 완료되면 게임 초기화 프로세스(`init()`)를 자동으로 트리거.
